주제: MVC 패턴 기반의 console 회원제 게시판
    1. 구현 기능
        회원      : 회원가입, 로그인, 아이디찾기, 비밀번호찾기, 회원탈퇴, 회원수정, [ex 직급(직급명, 호봉, 기본급여 등 지급 추가시에는 테이블 나누어야 하기도 함]
        게시판    : 글 작성, 글 전체 출력, 글 개별 출력, 글 삭제, 글 수정, 조회수
        댓글      : 댓글작성, 댓글출력, 댓글삭제, 댓글수정

    2. 데이터베이스 설계
        --- 샘플데이터 이용한 데이터 파악(머리에 안그려지면 눈에 보이도록 시각화 -- 협업에는 각 생각들이 다르기 때문에 추상화 -> 시각화 설계)
        1. 필드 파악: (영구저장 데이터를 파악) 아이디, 비밀번호, 이름, 연락처, 가입일, 글제목, 글내용,
                    글작성일, 글작성자, 글조회수, 댓글내용, 댓글작성일
                    - 비밀번호 확인[x], 총합계[x] 등 유효성검사, 연산데이터 데이터베이스에 저장 안함
        2. 필드 (기능)분류: 필드들을 기능별로 테이블 형식으로 분류, 중복 최소화 분류, 로그/기록 따른 분류
                    회원: 아이디, 비밀번호, 이름, 연락처, 가입일, 직급(직급만 넣을거면 분류할 필요X, 호봉, 기본급여 등이 있으면 따로 테이블)
                    게시물: 글제목, 글내용, 글작성일, 글작성자, 글조회수
                    댓글: 댓글내용, 댓글작성일, 댓글작성자
        3. 필드 관계
            회원 게시판 관계   : 회원이 게시물 작성, 회원 1명 - 게시물 1개, 회원1명 - 게시물 N개
            회원 댓글 관계    : 회원이 댓글 작성, 회원1명 - 댓글 1개, 회원1명 - 댓글 N개
            게시판 댓글 관계   : 게시물 안에 댓글 작성, 게시물1개 - 댓글1개, 게시물1개 - 댓글 N개
             * PK - FK : 1. PK N - FK
             * 관례적으로 테이블 1개당 PK1개
             회원번호PK -- 작성자(회원번호FK)
             회원번호PK -- 댓글작성자(회원번호FK)
             글번호PK -- 글번호(글번호FK)
        4. SQL 작성, 샘플

    3. 자바 프로젝트 패키지 구조 설계, 분류: 1. MVC기반 2. 기능별
    view 패키지            : view 클래스들을 보관하는 패키지, 역할: 입출력
        MemberView 클래스
        BoardView 클래스
    controller 패키지      : controller 클래스들을 보관하는 패키지, 역할: 데이터전달, 유효성검사, 데이터형변환
        MemberController 클래스
        BoardController 클래스
    model 패키지           : dto, dao 클래스들을 보관하는 패키지, 역할: 데이터 관리
        dto 패키지
            MemberDto 클래스
            BoardDto 클래스
            ReplyDto 클래스
        dao 패키지
            MemberDao 클래스
            BoardDao(=ReplyDao) 클래스
    AppStart 클래스        : 해당 프로그램을 시작하는 클래스

- 확인비밀번호를 입력받아 현재 로그인된
